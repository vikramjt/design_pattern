# Abstract Factory Pattern

## Working

Creates families of related objects without exposing the concrete classes to the client.

## Advantages

- Keeps related products consistent
- Hides concrete creation logic
- Supports interchangeable families

## Disadvantages

- Adds more abstractions
- Hard to extend with a new product type

## Usage

- Theme or platform specific object creation
- UI widgets
- Database driver families

## Example

`App.java` is the module entry point.
