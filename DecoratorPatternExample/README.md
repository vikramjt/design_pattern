# Decorator Pattern

## Working

Wraps an object with extra behavior without changing the original class.

## Advantages

- Flexible behavior extension
- Avoids subclass explosion
- Composable at runtime

## Disadvantages

- Can create many small objects
- Harder to trace behavior

## Usage

- Logging wrappers
- I/O streams
- Feature toggles

## Example

`App.java` is the module entry point.
