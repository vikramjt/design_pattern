# Builder Pattern

## Working

Builds complex objects step by step and keeps construction separate from representation.

## Advantages

- Clear object construction
- Works well with many optional fields
- Supports immutable objects

## Disadvantages

- Adds builder classes
- Can be excessive for simple objects

## Usage

- DTOs with many parameters
- Immutable domain objects
- Fluent configuration APIs

## Example

`App.java` is the module entry point.
