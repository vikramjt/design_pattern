# Flyweight Pattern

## Working

Shares common intrinsic state across many small objects to save memory.

## Advantages

- Lower memory usage
- Efficient for many similar objects
- Better sharing of immutable state

## Disadvantages

- More complex state management
- External state handling is tricky

## Usage

- Text rendering
- Game entities
- Large object catalogs

## Example

`App.java` is the module entry point.
