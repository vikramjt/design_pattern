# Prototype Pattern

## Working

Creates new objects by cloning an existing prototype instead of building from scratch.

## Advantages

- Fast object creation
- Useful for complex objects
- Reduces repeated initialization

## Disadvantages

- Deep copy can be tricky
- Cloning may be error-prone

## Usage

- Object templates
- Expensive initialization cases
- Copying preconfigured objects

## Example

`App.java` is the module entry point.
