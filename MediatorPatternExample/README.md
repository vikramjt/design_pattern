# Mediator Pattern

## Working

Centralizes communication between objects through a mediator instead of direct references.

## Advantages

- Reduces many-to-many dependencies
- Simplifies object collaboration
- Encapsulates interaction rules

## Disadvantages

- Mediator can become complex
- Centralized logic may become a bottleneck

## Usage

- Chat systems
- UI dialog coordination
- Workflow orchestration

`App.java` is the module entry point.
