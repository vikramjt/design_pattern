# Facade Pattern

## Working

Provides a simple interface to a complex subsystem.

## Advantages

- Easier client usage
- Reduces coupling to subsystem details
- Improves readability

## Disadvantages

- Can hide useful subsystem features
- May become a thin wrapper if poorly designed

## Usage

- Service orchestration
- Complex library wrapping
- Simplified API layers

## Example

`App.java` is the module entry point.
