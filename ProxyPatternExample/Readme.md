# Proxy Pattern

## Working

Uses a stand-in object to control access to another object.

## Advantages

- Adds access control
- Supports lazy loading
- Can isolate remote access complexity

## Disadvantages

- Adds indirection
- Can make debugging harder

## Usage

- Security proxies
- Lazy-loaded resources
- Remote service wrappers

## Example

`App.java` is the module entry point.
