# State Pattern

## Working

Changes behavior when the internal state of an object changes.

## Advantages

- Replaces large conditionals
- Keeps state-specific logic isolated
- Easy to extend with new states

## Disadvantages

- More classes
- State transitions can be harder to trace

## Usage

- Workflow engines
- Device state machines
- Order lifecycles

## Example

`App.java` is the module entry point.
