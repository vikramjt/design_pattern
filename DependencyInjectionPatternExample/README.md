# Dependency Injection Pattern

## Working

Supplies dependencies from the outside instead of creating them inside the class.

## Advantages

- Loose coupling
- Easier testing
- Better separation of concerns

## Disadvantages

- More wiring/configuration
- Can be harder to trace object creation

## Usage

- Framework-based applications
- Service wiring
- Test-friendly architecture

## Example

`App.java` is the module entry point.
