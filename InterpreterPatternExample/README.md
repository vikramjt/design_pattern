# Interpreter Pattern

## Working

Represents a grammar and evaluates expressions in that grammar.

## Advantages

- Easy for simple DSLs
- Clear grammar structure
- Good for repeated expression evaluation

## Disadvantages

- Class explosion for large grammars
- Not ideal for complex languages

## Usage

- Rule engines
- Small expression languages
- Parsing simple DSLs

## Example

`App.java` is the module entry point.
